The problem

You prompt. You wait. It ships.

Then nobody — including you — remembers why the code looks the way it does. Reviewers can only react to a 500-line PR. Refactors break invariants nobody wrote down.

Vibe Coding → technical debt nobody reviewed.

Plan Mode and SpecKit attack this at two different timescales.

Over time · for the life of a feature

SpecKit & SDD

Spec-Driven Development. Your intent lives in the repo as markdown — reviewable in PRs, regenerable, refactor-safe.

  • ✔ Constitution → project principles
  • ✔ Spec → what & why
  • ✔ Plan → how (stack)
  • ✔ Tasks → ordered steps
  • ✔ Implement → code, generated from the above

The SpecKit pipeline

Five steps. Each one a markdown artifact in git.

/speckit.constitution ⚖ Constitution memory/constitution.md
/speckit.specify 📋 Specify spec.md
/speckit.plan 🛠 Plan plan.md
/speckit.tasks ✅ Tasks tasks.md
/speckit.implement 💻 Implement app/ + tests/

Each artifact is derived from the previous one. Change the spec, regenerate the plan, regenerate the tasks, regenerate the code. Reproducible builds for product intent.
